# 国际化与本地化 (I18n and L10n)

I18n（国际化 (internationalization) 的缩写）是一种设计程序以支持多种语言的方式。L10n（本地化 (localization) 的缩写）是将文本翻译成用户语言的过程。Minecraft 使用 `Component`s 来实现这些。

## `Component`s

`Component` 是一段带有元数据的文本，元数据包括文本格式等。它可以通过以下方式之一创建（以下所有都是 `Component` 接口中的静态方法）：

| 方法 (Method) | 描述 (Description) |
|----------------|-------------------------------------------------------------------------------------------------------|
| `empty` | 创建一个空组件。 |
| `literal` | 创建一个包含给定文本的组件，并直接显示该文本而不进行翻译。 |
| `nullToEmpty` | 当给定 null 时创建一个空组件，否则创建一个字面量组件。 |
| `translatable` | 创建一个可翻译组件。给定的字符串将被解析为翻译键（见下文）。 |
| `keybind` | 创建一个包含给定按键绑定（已翻译）显示名称的组件。 |
| `nbt` | 创建一个表示给定路径 [NBT][nbt] 的组件。 |
| `score` | 创建一个包含记分板目标值 (scoreboard objective value) 的组件。 |
| `selector` | 为给定的[实体选择器 (entity selector)][selector]创建一个包含[实体 (entity)]名称列表的组件。 |
| `object` | 创建一个包含任意字体的组件，如图集精灵 (atlas sprite)。 |

`Component.translatable()` 还有一个可变参数 (vararg) 参数，用于接受字符串插值元素。这类似于 Java 的 `String#format`，但始终使用 `%s` 而不是 `%i`、`%d`、`%f` 和其他格式说明符，并在需要时调用 `#toString()`。

每个 `Component` 都可以使用 `#getString()` 来解析。解析通常是惰性的 (lazy)，这意味着服务器可以指定一个 `Component`，将其发送给客户端，然后每个客户端将自行解析 `Component`s（不同的语言可能导致不同的文本）。Minecraft 中的许多地方也会直接接受 `Component`s 并为您处理解析。

:::caution
切勿让服务器翻译 `Component`。始终将 `Component`s 发送到客户端并在那里解析它们。
:::

### `MutableComponent`

所有构造的组件通常都是 `MutableComponent`s。`MutableComponent` 提供了添加新组件同级 (siblings)（附加到此组件末尾）和设置文本[样式 (style)]的方法。构造或修改组件应始终引用 `MutableComponent`s。

### 文本格式化 (Text Formatting)

`Component`s 可以使用 `Style`s 进行格式化。`Style`s 是不可变的 (immutable)，修改时会创建一个新的 `Style` 对象，因此可以一次创建然后根据需要重用。

:::note
样式只能使用 `MutableComponent` 上的方法设置，因此请确保检查 `Component` 是否为 `MutableComponent`。
:::

`Style.EMPTY` 通常可以用作基础。多个 `Style` 可以使用 `Style#applyTo(Style other)` 进行合并，该方法返回一个新的 `Style`，其设置取自调用 `applyTo()` 方法的 `Style`，除非相应的设置不存在，在这种情况下使用作为参数传入的 `Style` 的设置。然后可以将 `Style`s 应用到组件，如下所示：

```java
MutableComponent text = Component.literal("Hello World!");

// 创建新样式。
Style blue = Style.EMPTY.withColor(0x0000FF);
// 样式使用类似构建器 (builder) 的模式。
Style blueItalic = Style.EMPTY.withColor(0x0000FF).withItalic(true);
// 除了斜体，我们还可以使样式加粗 (bold)、下划线 (underlined)、删除线 (strikethrough) 或模糊 (obfuscated)。
Style bold          = Style.EMPTY.withBold(true);
Style underlined    = Style.EMPTY.withUnderlined(true);
Style strikethrough = Style.EMPTY.withStrikethrough(true);
Style obfuscated    = Style.EMPTY.withObfuscated(true);
// 让我们将一些样式合并在一起！
Style merged = blueItalic.applyTo(bold).applyTo(strikethrough);

// 在组件上设置样式。
text.setStyle(merged);
// 将新样式合并到其中。
text.withStyle(Style.EMPTY.withColor(0xFF0000));
```

另一种更复杂的格式化选项是使用点击 (click) 和悬停 (hover) 事件：

```java
// 点击事件总共有 8 个选项，悬停事件总共有 3 个选项。
ClickEvent clickEvent;
HoverEvent hoverEvent;

// 点击时在默认浏览器中打开给定 URL。
clickEvent = new ClickEvent.OpenUrl(URI.create("http://example.com/"));
// 点击时打开给定文件。出于安全原因，这不能从服务器发送。
clickEvent = new ClickEvent.OpenFile("C:/example.txt");
// 点击时运行给定命令。
clickEvent = new ClickEvent.RunCommand("/gamemode creative");
// 点击时在聊天中建议给定命令。
clickEvent = new ClickEvent.SuggestCommand("/gamemode creative");
// 点击时更改书页。在书籍屏幕上下文之外不相关。
clickEvent = new ClickEvent.ChangePage("1");
// 将给定文本复制到剪贴板。
clickEvent = new ClickEvent.CopyToClipboard("Hello World!");
// 打开一个对话框屏幕（假设我们有一些 Holder<Dialog> dialog）
clickEvent = new ClickEvent.ShowDialog(dialog);
// 向服务器发送标识符和有效负载（默认情况下，记录发送的操作和有效负载）
clickEvent = new ClickEvent.Custom(ResourceLocation.fromNamespaceAndPath("examplemod", "custom"), Optional.empty());

// 悬停时显示给定的组件。也可以被格式化。
// 请注意，点击或悬停事件在悬停提示框中不起作用。
hoverEvent = new HoverEvent.ShowText(Component.literal("Hello World!"));
// 悬停时显示给定物品堆的完整提示框。
hoverEvent = new HoverEvent.ShowItem(new ItemStack(...));
// 悬停时显示给定实体的完整提示框。
// 参见 EntityTooltipInfo 的可能构造函数。
hoverEvent = new HoverEvent.ShowEntity(new HoverEvent.EntityTooltipInfo(...));

// 将事件应用到样式。
Style clickable = Style.EMPTY.withClickEvent(clickEvent);
Style hoverable = Style.EMPTY.withHoverEvent(hoverEvent);
```

甚至组件使用的字体也可以设置：

```java
// 设置样式要使用的字体。
// 必须匹配可用的 FontDescription.Resource（位于 `assets/<namespace>/font/<path>.json`）

// 指向 `assets/minecraft/font/illageralt.json`
ResourceLocation fontLocation = ResourceLocation.withDefaultNamespace("illageralt");

// 将字体应用到样式
Style customFont = Style.EMPTY.withFont(new FontDescription.Resource(fontLocation));
```

:::warning
除了 `FontDescription.Resource` 之外的其他字体描述类型在样式中使用时将抛出异常。要使用其他或自定义的 `FontDescription`s，请创建并向组件提供一个带有 `ObjectContents` 的 `ObjectInfo`。
:::

## 语言文件 (Language Files)

语言文件是 JSON 文件，包含从翻译键（见下文）到实际名称的映射。它们位于 `assets/<modid>/lang/language_name.json`。例如，ID 为 `examplemod` 的模组的美国英语翻译将位于 `assets/examplemod/lang/en_us.json`。Minecraft 支持的完整语言列表可以在[这里][mcwikilang]找到。

语言文件通常如下所示：

```json
{
    "translation.key.1": "Translation 1",
    "translation.key.2": "Translation 2"
}
```

### 翻译键 (Translation Keys)

翻译键是翻译中使用的键。在许多情况下，它们遵循格式 `registry.modid.name`。例如，ID 为 `examplemod` 的模组提供了一个名为 `example_block` 的方块，可能希望为键 `block.examplemod.example_block` 提供翻译。但是，您基本上可以使用任何字符串作为翻译键。

如果所选语言中没有翻译键的关联翻译，游戏将回退到美国英语 (`en_us`)（如果该语言不是所选语言的话）。如果美国英语也没有翻译，翻译将静默失败，并显示原始翻译键。

Minecraft 中的某些地方为您提供了获取翻译键的辅助方法。例如，方块和物品都提供了 `#getDescriptionId` 方法。对于物品，这些不仅可以查询，还可以通过 `Item.Properties#overrideDescription` 在需要时更改。如果物品根据其底层[数据组件 (data components)][datacomponent] 具有不同的名称，则可以通过在 `ItemStack` 上设置带有所需可翻译组件的 `CUSTOM_NAME` 数据组件来覆盖。还有一个 `Item#getName` 的变体，它接受一个 [`ItemStack`][itemstack] 参数来设置物品的默认组件。另一方面，`BlockItem`s 通过调用 `Item.Properties#useBlockDescriptionPrefix` 来设置描述 ID。`ResourceLocation`s 还提供了一种使用 `toLanguageKey` 构造翻译键的方法，该方法接受部分 (section) 或注册表 (registry) 以及可选的后缀。

:::tip
翻译键的唯一目的是用于本地化。不要将它们用于游戏逻辑，那是[注册名 (registry names)][regname] 的用途。
:::

### 翻译模组元数据 (Translating Mod Metadata)

从 NeoForge 20.4.179 开始，翻译文件可以使用以下键覆盖[模组信息 (mod info)][modstoml] 的某些部分（其中 `modid` 应替换为实际的模组 ID）：

| | 翻译键 (Translation Key) | 覆盖内容 (Overriding) |
|--------------|----------------------------------------|------------------------------------------------------------------------------|
| 描述 (Description) | `fml.menu.mods.info.description.modid` | 可以在 `[[mods]]` 部分中放置一个名为 `description` 的字段。 |

### 数据生成 (Datagen)

语言文件可以[数据生成 (datagenned)][datagen]。为此，请扩展 `LanguageProvider` 类并在 `addTranslations()` 方法中添加您的翻译：

```java
public class MyLanguageProvider extends LanguageProvider {

    public MyLanguageProvider(PackOutput output) {
        super(
            // 由 `GatherDataEvent.Client` 提供。
            output,
            // 您的模组 ID。
            "examplemod",
            // 要使用的语言环境。您可以为不同的语言环境使用多个语言提供者。
            "en_us"
        );
    }
    
    @Override
    protected void addTranslations() {
        // 添加具有给定键和给定值的翻译。
        this.add("translation.key.1", "Translation 1");
        
        // 为各种常见对象类型提供了辅助方法。大多数辅助方法有两个变体：一个用于对象本身的 add() 变体，
        // 以及一个接受对象供应器 (supplier) 的 addTypeHere() 变体。
        // 由于泛型类型擦除，供应器变体的不同名称是必需的。
        // 以下所有示例都假设存在所需类型的供应器的值。

        // 添加方块翻译。
        this.add(MyBlocks.EXAMPLE_BLOCK.get(), "Example Block");
        this.addBlock(MyBlocks.EXAMPLE_BLOCK, "Example Block");
        // 添加物品翻译。
        this.add(MyItems.EXAMPLE_ITEM.get(), "Example Item");
        this.addItem(MyItems.EXAMPLE_ITEM, "Example Item");
        // 添加物品堆翻译。这主要用于具有 NBT 特定名称的物品。
        this.add(MyItems.EXAMPLE_ITEM_STACK.get(), "Example Item");
        this.addItemStack(MyItems.EXAMPLE_ITEM_STACK, "Example Item");
        // 添加实体类型翻译。
        this.add(MyEntityTypes.EXAMPLE_ENTITY_TYPE.get(), "Example Entity");
        this.addEntityType(MyEntityTypes.EXAMPLE_ENTITY_TYPE, "Example Entity");
        // 添加状态效果翻译。
        this.add(MyMobEffects.EXAMPLE_MOB_EFFECT.get(), "Example Effect");
        this.addEffect(MyMobEffects.EXAMPLE_MOB_EFFECT, "Example Effect");
        // 添加标签键翻译。
        this.add(MyTags.EXAMPLE_TAG, "Example Tag");
        // 添加维度翻译。
        this.add(MyDimensions.EXAMPLE_DIMENSION_KEY, "Example Dimension");
        // 添加生物群系翻译。
        this.add(MyBiomes.EXAMPLE_BIOME_KEY, "Example Biome");
    }
}
```

然后，像在 `GatherDataEvent.Client` 中注册任何其他提供者一样注册该提供者。

```java
@SubscribeEvent // 在模组事件总线上
public static void onGatherData(GatherDataEvent.Client event) {
    // 如果要添加数据包对象，请首先调用 event.createDatapackRegistryObjects(...)

    event.createProvider(MyLanguageProvider::new);
}
```
